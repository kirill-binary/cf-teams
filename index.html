What is cross-functional team?

A cross-functional team is a group of people with different functional expertise working toward a common goal. Team members must have all of the skills necessary to create an increment of work.

Our components teams:

  • Frontend department
  • Backend department
  • DBA department
  • Quality Assurance Department
  • DevOps Department

Lets take a look at work flow:

  • Request for feature
  • BE writing code
  • BE and DBA review
  • BE rewrite DB part
  • Again review
  • QA test
  • BE fix bugs
  • QA testing
  • Relise BE core
  • FE write code
  • FE request new fuctionality from BE
  • BE writing code
  • BE reviewing
  • QA testing
  • Relise BE code
  • FE writing a code
  • FE reviewing
  • QA testing
  • Relise FE code

Problems with components teams:

  • Inefficient communication
  • Each team has own priorities
  • Complicated planning

Project cross-functional teams:

Practice of assembling project teams using members of the organization from different functions. Typically, this would involve selecting a number of specialists under a generalist project manager.
